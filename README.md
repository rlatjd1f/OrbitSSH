# 🛰️ Orbit SSH

Orbit SSH는 여러 SSH 서버와 로컬 터미널을 한곳에서 관리하기 위한 macOS 데스크톱 애플리케이션입니다. 서버를 폴더 트리로 정리하고, 같은 서버를 여러 탭으로 열거나 하나의 탭 안에서 최대 4개의 터미널을 분할해 사용할 수 있습니다.

## ✨ 주요 기능

- 폴더 트리 기반 SSH 서버 관리
- 비밀번호 및 SSH 개인 키 인증
- macOS Keychain을 이용한 비밀번호 보관
- 로컬 터미널 기본 실행
- 동일 서버의 다중 탭 실행
- 탭 내부 최대 4개 터미널 분할
  - 2~3개: 가로 열 배치
  - 4개: 2×2 격자 배치
- 터미널 출력과 스크롤 기록 유지
- 종료된 터미널의 빠른 재접속
- 터미널 폰트, 글꼴 크기, 줄 간격 및 스크롤 버퍼 설정
- SSH KeepAlive 설정
- 드래그한 터미널 텍스트 자동 복사
- 한국어·영어 UI 지원
- GitHub 릴리즈 기반 새 버전 자동 확인 및 자동 업데이트

## 📋 지원 환경

- macOS
- SSH 접속에는 macOS에 기본 포함된 OpenSSH를 사용합니다.

## 🚀 설치

GitHub Releases에서 본인 Mac에 맞는 DMG 파일을 내려받아 설치합니다.

- **Apple Silicon Mac:** M1, M2, M3, M4 등 Apple 칩을 사용하는 Mac은 `OrbitSSH-<버전>-arm64.dmg`를 받습니다.
- **Intel Mac:** Intel 프로세서를 사용하는 Mac은 `OrbitSSH-<버전>-x64.dmg`를 받습니다.
- `Source code (zip)`과 `Source code (tar.gz)`는 개발용 소스 코드이며 일반 앱 설치 파일이 아닙니다.

Mac 종류는 화면 왼쪽 위의 **Apple 메뉴() → 이 Mac에 관하여**에서 확인할 수 있습니다. `칩` 항목에 Apple M 시리즈가 표시되면 Apple Silicon이고, `프로세서` 항목에 Intel이 표시되면 Intel Mac입니다.

## ⚠️ 처음 실행할 때 macOS 경고가 보이는 경우

현재 공개 릴리스는 Apple Developer ID 공증이 적용되지 않았으므로 처음 실행할 때 macOS Gatekeeper 경고가 표시될 수 있습니다.

Chrome 또는 Safari로 받은 앱에서 **“Orbit SSH은(는) 손상되었기 때문에 열 수 없습니다”** 메시지가 표시되면, 앱을 Applications 폴더로 옮긴 뒤 터미널에서 다음 명령을 실행합니다.

```bash
xattr -dr com.apple.quarantine "/Applications/Orbit SSH.app"
open "/Applications/Orbit SSH.app"
```

이 명령은 앱 파일을 수정하지 않고 macOS가 다운로드 파일에 붙인 quarantine 플래그만 제거합니다.

## 🔐 SSH 연결 등록

1. 왼쪽 사이드바에서 **새 폴더**를 눌러 서버를 분류할 폴더를 만듭니다.
2. **새 커넥션**을 누릅니다.
3. 장비 이름, 호스트, 포트와 사용자를 입력합니다.
4. 비밀번호 또는 SSH 개인 키 인증을 선택합니다.
5. **추가**를 눌러 저장합니다.
6. 사이드바의 서버를 더블 클릭해 새 터미널 탭을 엽니다.

비밀번호 인증을 선택하면 비밀번호는 연결 설정 파일에 기록하지 않고 macOS Keychain의 `OrbitSSH` 서비스에 저장합니다. 개인 키 경로를 비워두면 SSH Agent 또는 OpenSSH 기본 키를 사용합니다.

## 🖥️ 터미널 사용법

- 앱을 열면 기본으로 로컬 터미널 세션이 열립니다.
- 같은 서버를 반복해서 더블 클릭하면 매번 독립적인 새 탭이 열립니다.
- `⌘ Cmd+D`를 누르면 현재 탭 안에 터미널이 하나씩 추가됩니다.
- 분할 터미널은 탭당 최대 4개까지 열 수 있습니다.
- 분할된 화면을 클릭하면 해당 터미널이 활성화됩니다.
- 연결이 종료된 화면에서 `Enter`를 누르면 같은 패널에서 다시 접속합니다.
- 터미널 텍스트를 드래그해 선택하면 별도 복사 단축키를 누르지 않아도 선택한 내용이 클립보드에 자동 복사됩니다.

## ⌨️ 단축키

| 단축키 | 동작 |
| --- | --- |
| `Ctrl+Tab` | 다음 탭으로 이동하고 터미널에 포커스 |
| `⌘ Cmd+D` | 현재 탭 내부에 터미널 분할 추가 |
| `Ctrl+[` | 이전 분할 터미널로 이동 |
| `Ctrl+]` | 다음 분할 터미널로 이동 |
| `Ctrl+T` | 현재 활성 SSH 세션과 같은 서버를 새 탭으로 열기 |
| `Ctrl+N` | 새 세션 선택 팝업 열기 |
| `⌘ Cmd+W` / `Ctrl+W` | 활성 분할 패널 닫기, 마지막 패널이면 탭 닫기 |
| `Ctrl+C` | 원격 터미널의 현재 입력 또는 작업 중단 |
| `Ctrl+D` | 원격 셸에 EOF 전달 |
| `⌘ Cmd+,` | 전역 설정 열기 |

## ⚙️ 설정

`⌘ Cmd+,`를 눌러 다음 전역 설정을 변경할 수 있습니다.

- 언어: 한국어 또는 English
- 터미널 폰트, 글꼴 크기와 줄 간격
- 커서 깜빡임
- 탭당 스크롤 버퍼 크기
- 기본 포트와 인증 방식
- SSH KeepAlive 간격
- 현재 앱 버전 확인 및 수동 업데이트 검사

언어를 변경하고 설정을 저장하면 사이드바, 연결 팝업, 터미널 안내, 업데이트 화면과 macOS 상단 메뉴가 선택한 언어로 함께 전환됩니다. 사용자가 등록한 장비 이름과 폴더 이름은 변경되지 않습니다.

## 🔄 앱 업데이트

Orbit SSH는 실행 시 GitHub에 공개된 최신 정식 버전을 확인합니다. 현재 설치된 앱보다 높은 버전이 공개되어 있으면 화면 오른쪽 아래에 업데이트 알림을 표시합니다.

- Apple Silicon Mac에서는 앱이 `arm64.dmg` 업데이트를 자동으로 선택합니다.
- Intel Mac에서는 앱이 `x64.dmg` 업데이트를 자동으로 선택합니다.
- 상단 메뉴의 **업데이트 확인…** 또는 **설정 → 앱 업데이트**에서 수동으로 업데이트를 다시 검사할 수 있습니다.
- **업데이트 설치**를 누르면 Orbit SSH가 백그라운드 임시 폴더에 설치 파일을 받은 뒤, 현재 앱을 새 버전으로 교체하고 자동으로 다시 시작합니다.

업데이트가 진행되는 동안 다운로드 진행률과 적용 중 애니메이션이 표시됩니다. 사용자가 DMG를 직접 열거나 Applications 폴더로 옮길 필요는 없습니다.

## 💾 데이터 저장 위치

연결과 설정은 macOS 사용자 데이터 디렉터리에 저장됩니다.

```text
~/Library/Application Support/Orbit SSH/connections.json
~/Library/Application Support/Orbit SSH/settings.json
```

비밀번호는 위 파일에 포함되지 않으며 macOS Keychain에 별도로 저장됩니다.
